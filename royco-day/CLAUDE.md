# royco-day

A Graph Protocol subgraph indexing **Royco Day**: 3-tranche markets (senior /
junior / liquidity). Deployed to **Goldsky**; its entities are mirrored into
**Neon Postgres** by a Goldsky **Pipeline**.

Status: **base setup only.** Every handler in `src/` is a wired, compiling
no-op stub. Writing them is the next phase.

Sibling `../royco-rwa` is the architectural reference. It is a good pattern for
structure and a bad one for tests (it has none) — copy its idioms, not its gaps.
Where this package deliberately diverges, it says so below.

---

## 1. The build loop

`subgraph.yaml` and `src/constants/static.ts` **do not exist as source**. They are
rendered by mustache from `config/markets/`. `graph codegen` reads
`subgraph.yaml` to know which ABIs to bind. Therefore:

> **codegen cannot run before prepare. Ever.**

```bash
npm run prepare:markets:mainnet   # mustache -> subgraph.yaml + static.ts + pipeline yaml
npm run codegen                   # subgraph.yaml + abis/ -> generated/
npm run build                     # generated/ + src/ -> build/   (also re-runs prepare)
npm test                          # node checks + matchstick
```

Or just `/verify`, which runs them in order and hands back a distilled error list
instead of hundreds of lines of `asc` output.

**`graph` and `mustache` are not on your PATH.** They are local devDependencies
and only resolve inside `npm run` scripts. `bash ./scripts/markets/*.sh` run
directly would die with `command not found` — those scripts use `npm exec` for
exactly this reason. For one-offs use `npx graph ...`.

## 2. Never edit these files

| File | Generated by | Edit this instead |
|---|---|---|
| `subgraph.yaml` | `npm run prepare:markets:*` | `config/markets/subgraph.template.yaml` |
| `src/constants/static.ts` | `npm run prepare:markets:*` | `config/markets/constants.template.ts` (values: `config/markets/networks/*.json`) |
| `config/markets/pipeline.template.yaml` | `npm run generate` | `schema.graphql` + `config/markets/pipeline.config.json` |
| `royco-day-markets-pipeline.yaml` | `npm run prepare:markets:*` | as above |
| `tests/generated/abi-signatures.ts` | `npm run generate` | `abis/*.json` |
| `generated/**`, `build/**` | `graph codegen` / `graph build` | nothing — gitignored |

The first two are **gitignored** (they're whichever network was prepared last —
a committed copy would lie about `CHAIN_ID`). The rest are **committed but
generated**: reviewable in a diff, reverted by the next `generate`.

If you are editing `subgraph.yaml`, you are in the wrong file. It builds, it
indexes, and the next `prepare` silently reverts you.

## 3. AssemblyScript is not TypeScript

`src/**` compiles to WASM via `asc`. It looks like TypeScript. It is not.

| Don't | Why | Do |
|---|---|---|
| `arr.find(x => x.id == t)` | **Closures are unsupported.** Any arrow fn capturing an outer variable fails with `ERROR AS100: Not implemented: Closures`. Kills `.map`/`.filter`/`.find`/`.some`/`.reduce` whenever the callback captures. | explicit `for (let i = 0; i < arr.length; i++)` |
| `try { } catch (e) { }` | No exception handling. | `try_foo()` on contract bindings (§5). Nothing else recovers. |
| `let x: string \| number` | No unions. The only legal one is `T \| null`, reference types only. | pick one type, or two fields |
| `if (x === undefined)` | `undefined` doesn't exist. | `null`, reference types only. `i32`/`BigInt` are never null. |
| `foo(...args)` | No spread. | pass positionally |
| `const o = { a: 1 }` | Object literals aren't structural types. | declare a `class` |
| `JSON.parse(s)` | Not available. | `json.fromBytes()` from `graph-ts` |
| `a + b` on `BigInt` | Overloads exist but break on `BigInt \| null` and read ambiguously. | `.plus()` `.minus()` `.times()` `.div()` `.pow(u8)` |
| `` `${a}_${b}` `` | Compiles, but house style across this monorepo is `.concat()`. Match it. | `a.concat("_").concat(b)` |
| `bytes as Address` | `as` is a static cast; it won't reinterpret. | `changetype<Address>(bytes)` |
| `addr.toString()` | On `Address`/`Bytes` this yields garbage (UTF-8 over raw bytes). | **`.toHexString()`** — always, for every address/hash into a `String!` |
| `BigInt.fromI32(10).pow(decimals)` | `.pow()` takes `u8`; `decimals` is `i32`. | `.pow(u8(decimals))` |

`!` non-null assertion works, but flow analysis usually makes it unnecessary:

```ts
let s = DayVaultState.load(id);
if (!s) {
  s = new DayVaultState(id);   // narrowed to non-null from here on
}
s.updatedAtBlockTimestamp = event.block.timestamp;   // legal, no `!`
```

Never null-check a non-null schema field (`if (!entity.someString)`) — a
`String!` is never null in AS, so the branch is dead code. royco-rwa has exactly
this dead check at `src/royco-vault-tranche-v2.ts:36`. Don't copy it.

## 4. ABI → AssemblyScript types

**The uint24/uint32 boundary is asymmetric, and it's the top build-breaker here** —
`RoycoDayAccountant` packs its 27-field state with exactly these types.
(Verified against `generated/`, not folklore.)

| Solidity | AS | into a `BigInt!` field |
|---|---|---|
| `int8`–`int32`, `uint8`–`uint24` | **`i32`** | `BigInt.fromI32(x)` **required** |
| `uint32`–`uint256`, `int40`–`int256` | `BigInt` | direct |
| `bool` | `boolean` | → `Boolean!` |
| `address` | `Address` | `.toHexString()` → `String!` |
| `bytes`, `bytesN` | `Bytes` | `.toHexString()` → `String!` |
| `string` | `string` | direct |

`int32 → i32` but `uint32 → BigInt`. Not a typo — unsigned 32 doesn't fit signed i32.

Confirmed in this package's own generated code:

| `Accountant.getState()` member | Solidity | AS |
|---|---|---|
| `fixedTermDurationSeconds` | `uint24` | **`i32`** → needs `BigInt.fromI32()` |
| `lastMarketState` | `uint8` enum | **`i32`** → map `0`→`"fixed"`, `1`→`"perpetual"` |
| `fixedTermEndTimestamp` | `uint32` | `BigInt` → direct |
| `lastYieldShareAccrualTimestamp` | `uint32` | `BigInt` → direct |
| `twJTYieldShareAccruedWAD` | `uint192` | `BigInt` → direct |

`decimals()` is `uint8` → `i32` → assign **direct** to `Int!` (`Int!` *is* i32).
Never `BigInt.fromI32()` into an `Int!`.

Don't memorise this — ask the **abi-oracle** subagent.

## 5. Contract calls

**A raw call that reverts kills the handler and stalls the subgraph.** There is no
try/catch. `try_foo()` returns `{ value, reverted }` and never throws.

- **Use `try_`** for anything state-dependent: `previewSyncTrancheAccounting`,
  `convertToAssets`, `previewRedeem`, `totalAssets`, `getRawNAV`, `maxDeposit`.
  A 3-tranche market with coverage/liquidity invariants *will* revert in edge
  states (paused, fixed-term ended, coverage liquidation). Assume it does.
- **Raw is fine** for immutable metadata read once at deployment: `asset()`,
  `decimals()`, `symbol()`, `KERNEL()`, `TRANCHE_TYPE()`.

```ts
const res = tranche.try_convertToAssets(oneShare);
if (!res.reverted) {
  s.sharePriceNAV = res.value.nav;
}
// no else — leave the previous value; do NOT write zero
```

### What codegen will and won't bind

`graph codegen` emits bindings for `view`, `pure`, `nonpayable`, **and**
`constant`. Only **`payable`** is excluded
(`graph-cli/dist/protocols/ethereum/codegen/abi.js:439`).

In these ABIs the *only* payable function is `upgradeToAndCall`, which nothing
would ever call. **So in practice every function you want is callable** —
including the six multi-asset previews (`previewDepositMultiAsset`,
`previewRedeemMultiAsset`, `ltPreviewDepositMultiAsset`,
`ltPreviewRedeemMultiAsset`, `previewAddLiquidity`, `previewRemoveLiquidity`).
They are `nonpayable`, and `graph codegen` binds them anyway — verified in
`generated/templates/RoycoLiquidityTranche/RoycoLiquidityTranche.ts:1175`.

> If you have heard that "nonpayable functions get no binding" — that is **false**
> for graph-cli 0.97.1. It's a common claim and it does not survive contact with
> the generated code.

That said, **prefer the event over the call** for multi-asset flows:
`MultiAssetDeposit(caller, receiver, stAssets, quoteAssets, ltAssetsMinted, shares)`
and `MultiAssetRedeem(caller, receiver, owner, shares, stClaims, quoteAssets)`
carry the same numbers the previews return — for free, with no eth_call, and as
the exact post-state at that log index. A `nonpayable` preview is nonpayable
*because it isn't a pure read*; simulating it via `eth_call` works but is both
slower and semantically shakier than reading the event the contract already
emitted. Use `try_` if you must call one.

## 6. Domain model

`RoycoFactory.MarketDeploymentCompleted(indexed address template, indexed address
deployer, DeploymentResult result)` deploys one market. `result` is an 8-tuple:

```
(seniorTranche, juniorTranche, liquidityTranche, kernel, accountant, ydm, ltYdm, extras)
```

- **`result.kernel` IS the marketId.** `DayMarketState.id = <CHAIN_ID>_<kernel>`.
- **`result.ydm` is the JUNIOR YDM** → `juniorTrancheYdmAddress`. `result.ltYdm`
  → `liquidityTrancheYdmAddress`. The field is named `ydm`, not `jtYdm`.
- `result.extras` is `bytes` → `creationEncodedData` via `.toHexString()`.
- Create **all 5** templates from this handler. Miss one and its events are
  silently never indexed — no error, just missing rows.

The **accountant address is not the marketId**, so accountant handlers must
resolve accountant → market (via `ACCOUNTANT.KERNEL()`, or a map entity like
royco-rwa's `AccountantMarketMap`). Kernel handlers can look up directly.

### `Claims` — the quintuple

**These are not ERC-4626 vaults.** On all three tranches:

```
convertToAssets(uint256) -> Claims      // NOT uint256
totalAssets()            -> Claims      // NOT uint256
previewRedeem(uint256)   -> Claims      // NOT uint256
previewDeposit(uint256)  -> uint256     // this one IS a scalar
```

`Claims = (stAssets, jtAssets, ltAssets, stShares, nav)`. That struct is *why*
six entities repeat the same five fields. The mapping is fixed:

| `Claims` | schema field |
|---|---|
| `.stAssets` | `seniorTrancheAssets` |
| `.jtAssets` | `juniorTrancheAssets` |
| `.ltAssets` | `liquidityTrancheAssets` |
| `.stShares` | `seniorTrancheShares` |
| `.nav` | `nav` |

It also arrives via events: `Redeem(sender, receiver, claims, shares)` (claims at
index **2**) and LT's `MultiAssetRedeem(caller, receiver, owner, shares,
stClaims, quoteAssets)` (stClaims at index **4** — after `shares`).

`DayVaultState`'s two quintuples are the same call at two inputs:
`convertToAssets(sharesTotalSupply)` and `convertToAssets(10 ** shareTokenDecimals)`.

### `TrancheState` — 18 fields

`Kernel.previewSyncTrancheAccounting(uint8 trancheType)` → `(TrancheState state,
Claims claims, uint256 totalTrancheShares)`. **Use this one**, not the
Accountant's `previewSyncTrancheAccounting(uint256 stRawNAV, uint256 jtRawNAV)`,
which makes you supply the very NAVs you're trying to read and returns no claims.

`state` is **tranche-independent**; only `claims`/`totalTrancheShares` vary by
`trancheType`. Read each tranche's own `TRANCHE_TYPE()` — never hardcode 0/1/2
(the ABI carries no enum names, so the numbering is an inference).

The same struct is the entire payload of
`Accountant.TrancheAccountingSynced(TrancheState)`. Prefer the event.

**Divergence trap:** `DayMarketState.marketState` comes from
`Accountant.getState().lastMarketState` — the *stored* value. The *live* one is
`previewSyncTrancheAccounting().state.marketState`. They disagree between syncs.
The schema deliberately stores the former.

## 7. Schema field names vs ABI members

Schema names are **descriptive**, ABI members are **abbreviated**, and the two
deliberately do not match. `schema.graphql` annotates every contract-sourced
field with `<- ABI: <member>` at the point of use — read it there rather than
guessing. The conventions:

| ABI | schema |
|---|---|
| `st*` | `seniorTranche*` |
| `jt*` | `juniorTranche*` |
| `lt*` | `liquidityTranche*` |
| `tw*` | `timeWeighted*` |
| `*` (address) | `*Address` |
| `jtCoinvested` | `isJuniorTrancheCoinvested` |

Two names are **actively misleading** and were kept deliberately (renaming a
column post-deploy is a migration — §9). Do not "fix" them:

- **`marketState`** ← `lastMarketState`. The `last` matters; see §6.
- **`maxJuniorTrancheYieldShareAccruedWAD`** ← `maxJTYieldShareWAD`, and
  **`maxLiquidityTrancheYieldShareAccruedWAD`** ← `maxLTYieldShareWAD`. Despite
  "Accrued", these are **configured caps** (`setMaxYieldShares` /
  `MaxYieldSharesUpdated`) — *not* accruals. The real accruals are the
  `timeWeighted*Accrued*` fields sitting right beside them. **Never average or
  compare the two.**

## 8. Entity IDs, immutability, timestamps

Every id has a generator in `src/utils/global.ts`. **Never inline an id.**

| Entity | id | immutable |
|---|---|---|
| `GlobalTokenTransfer` | `<CHAIN>_<TX>_<LOG_IDX>` | ✅ |
| `GlobalTokenActivity` | `<CHAIN>_<TX>_<LOG_IDX>_<VAULT>_<CAT>_<SUBCAT>_<TOKEN_IDX>` | ✅ |
| `DayMarketState` | `<CHAIN>_<KERNEL>` | ❌ |
| `DayVaultState` | `<CHAIN>_<VAULT>` | ❌ |
| `DayVaultStateHistorical` | `<CHAIN>_<VAULT>_<ENTRY_IDX>` | ✅ |
| `DayPositionState` | `<CHAIN>_<VAULT>_<ACCOUNT>` | ❌ |
| `DayPositionStateHistorical` | `<CHAIN>_<VAULT>_<ACCOUNT>_<ENTRY_IDX>` | ✅ |
| `DayFeeState` | `<CHAIN>_<VAULT>_<ACCOUNT>_<MAJOR>_<MINOR>` | ❌ |
| `DayFeeStateHistorical` | `<CHAIN>_<VAULT>_<ACCOUNT>_<MAJOR>_<MINOR>_<ENTRY_IDX>` | ✅ |

**The failure mode that matters:** an `@entity(immutable: true)` is **write-once**.
A second `.save()` on the same id is a fatal `entity already exists` **at index
time** — it passes `graph build` and dies in production. Likewise a non-null
field left unset is fatal at index time. **`graph build` passing proves almost
nothing**; that's why the test harness exists.

- Addresses in ids are lowercase-hex-with-0x, from `.toHexString()`. A
  checksummed address won't match on `load()` and silently creates a duplicate.
- Mutable: `load()` → if null `new` + seed **every** non-null field → mutate →
  `save()`.
- **Entry index cursor**: the `*State` parent owns `lastHistoricalEntryIndex`.
  Creation snapshot is entry `0`. Each later snapshot writes `+1`, then stores
  it. Never derive an entry index from block number or timestamp — two writes in
  one block collide. (royco-rwa's `PositionStateHistorical` uses
  `_<BLOCK_TIMESTAMP>` and has exactly that bug. Don't copy it.)

**`createdAt*` / `updatedAt*`:** set `createdAt*` **exactly once**, inside the
`if (!entity)` branch — never touch it again. Re-stamping it on update is the
most common silent corruption here: it builds fine, indexes fine, and quietly
destroys every cohort/age query in Neon. Set `updatedAt*` on every save,
including creation (where they're equal). Immutable entities carry only
`createdAt*`. All values come off the event.

## 9. Schema ↔ Postgres coupling

`schema.graphql` → Goldsky subgraph → Pipeline → Neon. Entity/field names become
table/column names in snake_case (`DayVaultStateHistorical` →
`day_vault_state_historical`; `seniorTrancheAssets` → `senior_tranche_assets`).

1. **A field rename is a Postgres column migration** — regenerate, re-apply the
   pipeline, backfill, and break every downstream consumer. Names are frozen
   after first deploy. Add a field; don't rename one.
2. **Never rename a schema field to satisfy a build error.** The error is in
   `src/`.
3. `pipeline.template.yaml` is generated from `schema.graphql`. Adding an entity
   = edit schema → `/sync-pipeline` → commit both. An entity missing from the
   pipeline indexes fine, deploys fine, and **silently never reaches Neon**.
   (royco-rwa declares 31 entities and wires 24, with no record of which gaps are
   deliberate. `config/markets/pipeline.config.json`'s `exclude` list exists so
   that can't happen here.)

### The frozen shared-table contract

`GlobalTokenTransfer` and `GlobalTokenActivity` are **byte-identical** to
royco-rwa's, on purpose. royco-rwa (5 pipelines), royco-usd and staked-royco-usd
**all sink these into the same Neon tables** — `public.global_token_transfer` /
`public.global_token_activity`, secret `NEON_ROYCO_ESTATE`. royco-day joins that
union.

Rows don't collide (ids are chain+tx+log scoped), but the **column set is a
cross-package contract**. Changing a field here breaks *their* pipelines.
`scripts/checks/schema.test.mjs` diffs against `../royco-rwa/schema.graphql` and
fails if they drift. If that test fails, reconcile — don't edit the assertion.

## 10. Adding a handler

Use `/add-handler <Contract> <EventName>`. By hand:

1. **Ask `abi-oracle`** for the yaml event signature, or copy it from
   `tests/generated/abi-signatures.ts`. **Never hand-transcribe tuple
   components** — order is load-bearing, and a wrong signature yields a topic0
   matching nothing: zero rows, zero errors, no clue.
2. Add the `eventHandlers:` entry to **`config/markets/subgraph.template.yaml`**
   — not `subgraph.yaml` (§2). Include `receipt: true` (house convention).
3. List the entity under that data source's `entities:`.
4. Every ABI you `.bind()` must be in that data source's `abis:` list, or you get
   a confusing `Cannot find name`.
5. Export `handleX` from the `file:` named for that data source. The three
   tranches have identical event shapes but **distinct generated classes**, so
   shared logic lives in `src/handlers/` and is re-exported from each entry file
   (`export { handleTransfer } from "./handlers/...";`).
6. Add a test — copy `tests/handlers/_TEMPLATE.md`.
7. `/verify`.

## 11. Deploys are human-gated

Never run `goldsky ...`, `npm run deploy:*`, `npm run delete:*`, or
`bash ./scripts/markets/*.sh`. These are denied in `.claude/settings.json`
deliberately: `goldsky pipeline apply` mutates production Neon.

Prepare the artifacts, run `/verify`, show the diff, and tell the user the exact
command. Let them run it.

**The factory isn't deployed yet.** `config/markets/networks/mainnet.json` holds
a placeholder address and `startBlock: 0`. Fill those in before any real deploy.
